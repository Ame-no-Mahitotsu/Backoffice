### Code review

**Implementation is minimal and correct.**
`.sticky-top` wraps `<header>` and `<nav>` in `base.html`. `position: sticky; top: 0` keeps the element in normal document flow — content is not shifted or overlapped on load. `<main>` retains `padding: 2rem` which provides the correct clearance.

**`z-index: 100` is appropriate.** No competing stacking contexts in the current templates. Conservative value that will not conflict with any future overlays unless they deliberately stack higher.

**Test is pragmatic.** Asserting the CSS string in the raw server-rendered HTML is the right level for an inline-style dashboard — a real browser rendering test would be overkill and would require tooling we don't have. The assertion correctly fails before the change and passes after.

**No structural HTML changes** beyond the wrapper `<div>` — existing nav tests (`test_base_template_contains_nav_tabs`, `test_base_template_has_nav_links`) remain valid and green.
