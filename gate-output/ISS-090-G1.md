## Task interpretation

Add `detect-secrets==1.5.0` (Dominick recommendation, ISS-093) as a pre-commit hook so that all future `git commit` operations automatically scan staged files for secrets. This requires:
1. Adding `pre-commit==4.5.1` and `detect-secrets==1.5.0` to `pyproject.toml` dev extras
2. Creating `.pre-commit-config.yaml` at repo root
3. Generating `.secrets.baseline` by scanning the repo and reviewing false positives
4. Installing the git hook via `pre-commit install`

## Files to be touched

- `pyproject.toml` — add `pre-commit==4.5.1` and `detect-secrets==1.5.0` to `[project.optional-dependencies] dev`
- `.pre-commit-config.yaml` — new file at repo root
- `.secrets.baseline` — new file at repo root (generated, then committed)

All within the devops component scope.

## Base branch

`main`

## Implementation order

1. Update `pyproject.toml`
2. Install packages (`pip install pre-commit==4.5.1 detect-secrets==1.5.0`)
3. Create `.pre-commit-config.yaml`
4. Generate `.secrets.baseline` (`detect-secrets scan > .secrets.baseline`), review false positives
5. `pre-commit install`
6. `pre-commit run --all-files` — verify all files pass

## Tests to write at G2

This is a DevOps/infrastructure ticket. There are no pytest-style unit tests for a pre-commit configuration file. However, G2 verification can be done structurally:

- **`test_pre_commit_config_exists_and_contains_detect_secrets`**: reads `.pre-commit-config.yaml`, asserts it exists and contains the detect-secrets hook ID.
- **`test_secrets_baseline_exists`**: asserts `.secrets.baseline` exists at repo root.
- **`test_dev_dependencies_include_pre_commit_and_detect_secrets`**: reads `pyproject.toml`, asserts both pinned versions are present in the dev extras.

Test file: `tools/tests/test_pre_commit_config.py` (new file).

These tests will be red before the config files are created (G2), then green after (G3).

## Risks / unknowns

- `detect-secrets scan` may flag strings in `docker-compose.yml` comments and test files — these are genuine false positives. The baseline handles them; the G3 output will document what was acknowledged.
- Pre-commit hooks run on commit, not on test — so `pre-commit run --all-files` at G3 is the proof of correctness, not a pytest assertion.
