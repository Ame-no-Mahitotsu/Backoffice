# ISS-094 — Gate G1 (Analysis & Plan)

## Ticket interpretation
Add a hard 2000-character cap on `--notes` (create subcommand) and `--note`
(update subcommand) inputs in `ticket.py`. Reject with a clear error message
when input exceeds 2000 chars. Silent truncation is not acceptable (AC1).

## Implementation plan
1. Add a module-level constant `_MAX_NOTE_LEN = 2000` in `ticket.py`.
2. Add a guard at the top of `handle_create`:
   ```python
   if args.notes and len(args.notes) > _MAX_NOTE_LEN:
       print(f"Error: --notes value exceeds {_MAX_NOTE_LEN}-character limit ...", file=sys.stderr)
       sys.exit(1)
   ```
3. Add an equivalent guard at the top of `handle_update` for `args.note`.
4. Write a new test file `tools/tests/test_ticket_note_length.py` with 4 tests:
   - `test_create_rejects_notes_exceeding_2000_chars` — 2001 chars → SystemExit
   - `test_create_accepts_notes_at_exactly_2000_chars` — 2000 chars → no error
   - `test_update_rejects_note_exceeding_2000_chars` — 2001 chars → SystemExit
   - `test_update_accepts_note_at_exactly_2000_chars` — 2000 chars → no error

## Files to be touched
- `ticket.py` — add `_MAX_NOTE_LEN` constant + validation guards in `handle_create` and `handle_update`
- `tools/tests/test_ticket_note_length.py` — new test file (4 tests)
- `memory/workspace/tooling/gate-output/ISS-094-G1.md` (this file)

## Risks and mitigations
- **Risk**: Accidentally breaking `handle_create` for the common empty-notes path.
  - **Mitigation**: Guard only fires when `args.notes` is non-empty and > 2000 chars.
- **Risk**: `handle_update` receives `args.note` as empty string (default) — must not reject.
  - **Mitigation**: Guard fires only when `len(args.note) > _MAX_NOTE_LEN` (empty string = 0 ≤ 2000).
- **Risk**: Violating AC4 — existing tests break.
  - **Mitigation**: All existing note values in tests are well under 2000 chars; guard is additive.

## Acceptance criteria mapping
- AC1: `sys.exit(1)` with stderr message on > 2000 chars — does not truncate
- AC2: Cap applied in both `handle_create` (--notes) and `handle_update` (--note)
- AC3: Unit tests assert error at 2001 chars, not at 2000 chars
- AC4: Full test suite passes — existing tests use notes << 2000 chars
