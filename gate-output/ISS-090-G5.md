**Non-blocking advisory — hook strategy deviation from ISS-093 spec:**
Dominick's recommendation (ISS-093) specified the remote hook pattern:
```yaml
repo: https://github.com/Yelp/detect-secrets
rev: v1.5.0
```
Salvatore used `repo: local` with `language: system` instead. This is a legitimate implementation choice (avoids pre-commit creating a second isolated env when detect-secrets is already in our venv), and it works — proven by the live commit test. However:
- It creates a hard dependency on the venv being active at commit time
- `pre-commit run --all-files` blocked in the dev environment (Norton 360 AV — documented in G3)
- If a future developer commits without activating `.venv`, `detect-secrets-hook` will not be found

This is an accepted trade-off given the project's single-operator nature and the mandatory venv rule in ai-standards.md. No change required; documenting for awareness.
