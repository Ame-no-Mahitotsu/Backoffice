# ISS-097 — Gate G1 (Analysis & Plan)

## Ticket interpretation
Add a ticket detail view to the PM Dashboard:
1. A new Flask route `GET /tickets/<ticket_id>` in `dashboard.py` that queries
   the SQLite DB for a single ticket and renders a detail template.
2. A new Jinja2 template `templates/ticket_detail.html` showing all ticket fields.
3. A "View" link/button in the tickets list (`templates/tickets.html`) per row.
4. 404 response when the ticket ID is not found in the DB.
5. A "Back" link on the detail page returning to `/tickets` (with filter params
   if the Referer header is available, otherwise plain `/tickets`).

## Implementation plan

### `dashboard.py`
```python
@app.route("/tickets/<ticket_id>")
def ticket_detail(ticket_id):
    db = _get_db()
    row = db.execute(
        "SELECT id, project, date, updated, type, component, title, tags, "
        "status, owner, urgency, severity, sprint, closed, blocks, blocked_by, notes "
        "FROM tickets WHERE id = ?",
        (ticket_id,)
    ).fetchone()
    if row is None:
        abort(404)
    back_url = request.referrer or "/tickets"
    return render_template("ticket_detail.html", active="tickets", ticket=row, back_url=back_url)
```

### `templates/ticket_detail.html`
New template: extends `base.html`. Displays all fields in a definition-list or
table layout. Includes a `<a href="{{ back_url }}">← Back</a>` link.

### `templates/tickets.html`
Add a "View" column to the existing ticket table, with a link:
`<a href="/tickets/{{ t['id'] }}">View</a>`

## Risks and mitigations
- **Risk**: Flask route conflict between `/tickets` and `/tickets/<ticket_id>`.
  - **Mitigation**: Flask routes are matched in order; `<ticket_id>` is a string
    converter that only matches when a path segment follows `/tickets/`. No conflict.
- **Risk**: `abort(404)` requires Flask import.
  - **Mitigation**: Already imported via `from flask import Flask, g, render_template, request`.
    Add `abort` to the import.
- **Risk**: `request.referrer` may be `None` in test environment.
  - **Mitigation**: `back_url = request.referrer or "/tickets"` safely falls back.
