### Code review

**Guards are correctly placed and logically sound.**
Both use the same pattern: `if args.notes and len(args.notes) > _MAX_NOTE_LEN`. The truthy short-circuit means `None` and `""` are never evaluated — no false rejects on optional fields. The error message includes the actual character count, which is useful for the caller to understand what to fix.

**`handle_close` is implicitly covered.**
`handle_close` constructs an `argparse.Namespace` and calls `handle_update` directly — the guard fires there for free. No duplication needed.

**Test helper pattern is reusable.**
`_create_args()` and `_update_args()` with `**kwargs` overrides is the right pattern for this kind of unit test. Clean, readable, minimal.

**One gap worth a follow-up:** `handle_reopen` writes a note through its own path without calling `handle_update`, so the cap does not apply there. Out of scope for this ticket's AC but worth raising as a separate hygiene item if `reopen` becomes part of automated pipelines.
