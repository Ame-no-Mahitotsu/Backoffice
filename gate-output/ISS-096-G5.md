### Findings

**Template logic is correct.**
Each of the 4 status cards uses `{% if status_counts["..."] > 0 %}` to conditionally wrap the count in an `<a>` tag. Zero counts render as plain text. AC1 and AC2 ✓

**HTML entity encoding is correct.**
`&amp;` used in the `href` attribute for the `&` separator: `/tickets?sprint={{ selected_sprint }}&amp;status=open`. This is the correct encoding inside an HTML attribute. The Flask test client will decode it correctly, and browsers will follow the link correctly. ✓

**Link styling is appropriate.**
`color: inherit` on `.status-card .count a` preserves the existing count colour. `text-decoration: none` with hover underline is the right UX pattern for a number that is a link. ✓

**Only 4 statuses displayed.**
The sprint panel shows `open`, `in-development`, `in-testing`, `resolved` — this is the panel's existing design from ISS-080. All 4 are handled with the conditional. This is not a gap introduced by ISS-096.

**Tests cover both sides of the conditional.**
`test_sprint_non_zero_status_count_is_a_link` — confirms the anchor is rendered when count > 0.
`test_sprint_zero_status_count_is_not_a_link` — confirms no anchor when count = 0. AC2 ✓
