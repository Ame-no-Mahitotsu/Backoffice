### Code review

**Route is correctly structured.**
`@app.route("/tickets/<ticket_id>")` — Flask's string converter only matches when a path segment follows `/tickets/`, so there is no ambiguity with the existing `/tickets` list route. `abort` correctly added to the Flask import.

**Query is safe.**
`db.execute("SELECT ... WHERE id = ?", (ticket_id,))` — parameterised, no interpolation. `fetchone()` returns `None` cleanly for a missing ID, and `abort(404)` fires immediately. ✓

**`back_url` fallback is correct.**
`request.referrer or "/tickets"` handles the test environment (no Referer header) and the browser case (preserves list URL with filters). Advisory: in a public-facing app, using `referrer` directly could enable open redirects — but this dashboard is internal only (Constitution §3 / §8), so the risk is nil.

**Template covers all 17 fields.**
`ticket_detail.html` renders all columns from the SELECT. `white-space: pre-wrap` on the notes cell is the right choice for the pipe-separated note entries. Status badge with inline Jinja2 CSS conditionals is functional and server-controlled — not a security concern.
