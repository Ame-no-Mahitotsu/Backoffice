### Code review

**Jinja2 conditional is correct and readable.**
The `{% if status_counts["open"] > 0 %}...<a href>...{% else %}{{ count }}{% endif %}` pattern is applied consistently to all four status cards. The approach is deliberate — no abstraction needed for four cards.

**`&amp;` encoding is correct.**
The `href` attribute uses `&amp;` as the query string separator, which is the correct HTML encoding for `&` inside an attribute. The test assertion accounts for both `&amp;` and bare `&` with an `or` — defensively correct, though only `&amp;` will appear in the rendered HTML.

**Tests are well-scoped.**
`test_sprint_non_zero_status_count_is_a_link` seeds a ticket with status `open`, fetches `/sprint?sprint=link-sprint`, and asserts the anchor href is present. `test_sprint_zero_status_count_is_not_a_link` seeds the same sprint with zero resolved tickets and confirms no resolved anchor. Both tests use isolated `tmp_path` DBs — no shared state.

**Only 4 statuses shown** — this is the sprint panel's existing design (from ISS-080). All 4 have been handled. Not a gap introduced here.
