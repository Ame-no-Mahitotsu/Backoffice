## Task interpretation

`_seed_messages()` in `tools/tests/test_dashboard_messages.py` inserts three message rows using `status='active'`, which is not a valid value in the `MessageStatus` enum (`open` and `closed` are the only valid values, per `tools/pm/ticket_schema.py`). Although the dashboard route renders the raw string and no runtime failure occurs today, the fixture is semantically incorrect and could silently mask a future validation bug. The fix is a one-line-per-row correction: replace all three `'active'` occurrences in the `_seed_messages()` INSERT data with `'open'`.

  - In `_seed_messages()`, change `status='active'` → `status='open'` for all three rows in the `executemany` value list (the tenth positional column in each tuple — positions line 22, 23, 24 of the INSERT data).

No other files are touched. AC3 explicitly prohibits changes to other test files.
