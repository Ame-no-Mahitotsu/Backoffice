## Task interpretation

The keyword search in `dashboard.py` builds a `WHERE` clause that includes `tags LIKE ?` alongside `title LIKE ?` and `notes LIKE ?` (line 76), meaning a ticket matched solely by its tags column should appear in results. However, the existing test suite only exercises keyword matching via title (`test_tickets_view_keyword_search_matches_title`) and notes (`test_tickets_view_keyword_search_matches_notes`). The gap is that no test seeds a ticket whose keyword appears exclusively in the `tags` column, leaving the tags branch of the SQL OR completely uncovered. This ticket adds two focused tests to close that gap.

## Existing test infrastructure (relevant to this ticket)

- **`seeded_client` fixture** (defined inline in `test_dashboard.py`, not in `conftest.py` which is empty): creates a fresh `tmp_path` db, calls `create_app(db_path=...)`, seeds three rows via `_seed_tickets()`, and returns `app.test_client()`. The seeded rows are ISS-001 (`tags="alpha"`, title "Alpha feature"), ISS-002 (`tags="beta"`), ISS-003 (`tags=""`).
- **`_seed_tickets(db_path, rows)`**: helper that inserts raw rows via `sqlite3` directly into the `tickets` table. Each row is built by merging `_BASE_ROW` defaults with per-test overrides using `{**_BASE_ROW, ...}`.
- **`_BASE_ROW`**: provides safe defaults for all non-nullable columns so tests only specify what they care about.
- **Inline tempdir pattern**: date-filter tests (`test_tickets_view_filter_date_from`, `test_tickets_view_filter_date_to`) bypass `seeded_client` and create their own isolated db using `tempfile.TemporaryDirectory()`. This is the correct pattern here too — the new tests need full control over what is and is not in the db to make the positive and negative assertions watertight.
- **`seeded_client` cannot be reused as-is for AC1**: ISS-001 has both `tags="alpha"` and `title="Alpha feature"`, so searching for "alpha" would match on title as well — that does not isolate the tags branch. A dedicated seed is required.

## Tests to write

1. **`test_tickets_view_keyword_search_matches_tags`**
   Seeds a ticket whose `tags` column contains the keyword (`"uniquetag"`) while `title` and `notes` contain no substring of that keyword. Asserts the ticket ID appears in the `/tickets?q=uniquetag` response body.

2. **`test_tickets_view_keyword_search_does_not_match_when_no_tag_title_notes`**
   Seeds a ticket whose `tags`, `title`, and `notes` all contain no substring of the search keyword (`"uniquetag"`). Asserts the ticket ID is absent from the `/tickets?q=uniquetag` response body.
