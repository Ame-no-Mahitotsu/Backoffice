## Task interpretation

`query_tickets` in `sqlite_reader.py` emits `field IN ()` when a list field is set to `[]`. SQLite rejects `IN ()` with a `ProgrammingError`. The CLI guards against this via `nargs=+`, but programmatic callers can still trigger it. Fix: treat an empty list identically to `None` — skip the filter.

## Files to be touched

- `tools/pm/sqlite_reader.py` — add `if not value: continue` guard before building `IN` clause
- `tools/tests/test_sqlite_reader.py` — add one test

## Fix

```python
if isinstance(value, list):
    if not value:          # empty list → treat as no filter
        continue
    placeholders = ...
```
