## Change implemented

Added two tests to `tools/tests/test_dashboard.py` covering the keyword search
behaviour against the `tags` column in the dashboard ticket list view:

- `test_tickets_view_keyword_search_matches_tags`: creates a ticket with a
  distinctive tag, performs a keyword search for that tag value, and asserts the
  matching ticket appears in the response.
- `test_tickets_view_keyword_search_does_not_match_when_no_tag_title_notes`:
  creates two tickets — one whose tag/title/notes contain none of the search
  keyword — and asserts that ticket does NOT appear in the response, confirming
  the filter excludes non-matching rows.

No production code was changed. The `tags LIKE ?` clause already existed in the
dashboard query; these tests provide coverage proof.
